# ADR-0003: Energy-Based Output Head for GNN Surrogate

* **Status:** Proposed — retraining required
* **Date:** 2026-04-22
* **Related:** ADR-0001 (GNN surrogate), ADR-0002 (h0 snapshot fix)
* **Plan:** `.claude/plans/floofy-leaping-quilt.md`
* **Branch:** `architecture/potential-head` (graphorge + torch-fem)

## Context and Problem Statement

After ADR-0002 fixed the Jacobian inconsistency from hidden-state
drift, 1x1 elastoplastic_nlh patches converge with the expected
quadratic-ish rate (114.8 → 3.39 → 7.67e-3 → 3.2e-7). **5x5 patches
still diverge**:

```
Inc 1: 76.87 → 176.87 → 272.12 → 779.97 → 3685 → 4821 → 5257 →
       4207 → 3502 → 3329 → 3062 → 4232 → NaN (singular K)
```

Extensive diagnostics on 2026-04-21/22 established that neither
h0-snapshot, post-hoc symmetrisation, nor SPD projection can rescue
the 5x5 surrogate. The model itself is the problem.

## Diagnostic Evidence

### Autodiff is correct — node ordering is correct

Finite-difference probe in `surrogate_integrate_material` (obs
#1948, #1972) comparing FD vs `jacfwd` Jacobian:

```
‖K_fd − K_ad‖ / ‖K_ad‖  ≈  5e-5     # relative agreement
```

The graph builder and node-ordering hypothesis (obs #1965) was
falsified by this probe. What `jacfwd` returns is the true
derivative of the surrogate's force output with respect to
boundary displacements.

### The surrogate learned a non-conservative force field

Measured on 5x5 elastoplastic_nlh patches, `F_GNN : R^{40} → R^{40}`:

| Metric                                 | Value       |
|----------------------------------------|-------------|
| `‖K − Kᵀ‖ / ‖K‖`                      | 0.71 – 0.86 |
| `‖F(u = 0)‖`                          | 9.61        |
| `‖K_sym − K_analytic‖ / ‖K_analytic‖` | 0.96        |
| fraction of negative eigenvalues of K  | ≈ 60%       |

Root cause: the training loss is MSE on forces. Supervising `F`
constrains the zero-order map `u → F(u)` but leaves the Jacobian
`∂F/∂u` free to be anything — including highly anti-symmetric and
indefinite. A physically consistent tangent requires
`K_ij = ∂F_i / ∂u_j = ∂F_j / ∂u_i`, i.e. `F` must be the gradient
of some scalar potential (Schwarz's theorem). No such constraint
exists in the current loss.

### Why 1x1 tolerates this and 5x5 does not

1x1 patches have 4 boundary nodes × 2 dims = 8 DOF → 8×8 K. At this
size the anti-symmetric component of the learned `K` can still be
dominated by its symmetric part near the elastic regime, and Newton
recovers. 5x5 patches have 20 boundary nodes × 2 dims = 40 DOF →
40×40 K. The pathology compounds and Newton diverges from the
first increment.

### Post-hoc patches — all fail

Tested in-place in `surrogate_integrate_material` and recorded in
claude-mem:

| Post-hoc correction                         | Inc 1 residual | Result  |
|---------------------------------------------|----------------|---------|
| raw (no correction)                         | 1.0e+02 → 5e+3 | diverge |
| `K ← (K + Kᵀ)/2` (symmetrise, obs #1953)   | 1.0e+02 → 5e+3 | diverge |
| symmetrise + eigenvalue clamp (SPD, #1954) | 1.0e+02 → 9e+3 | diverge |
| mean-force removal (obs #2091, aa9d504)    | no help        | diverge |

The reason is structural: when `K` is replaced with `K_fixed` but
`F` is left as `F_GNN`, the relation `K = ∂F/∂u` is **broken**. The
Newton step `Δu = −K_fixed⁻¹ F_GNN` no longer descends the residual
field generated by `F_GNN`. The model is internally inconsistent, and
the inconsistency cannot be removed by adjusting one side in
isolation.

## Decision Drivers

* `K` must be symmetric (Schwarz) — enforced structurally, not
  post-hoc.
* `K` should be positive definite in the elastic regime.
* `F` and `K` must satisfy `K = ∂F/∂u` exactly — no post-processing.
* Plasticity history (`h_{t-1}`) must participate in the prediction
  but must not leak gradients through BPTT (would break `K` symmetry
  across timesteps).
* Training must remain tractable — no 10x cost blow-up.
* The consumer side (`torch-fem/src/torchfem/base.py`) must be able
  to compute `F` and `K` from the new head at inference time.

## Considered Options

* **Option A:** Keep force-MSE training, apply symmetrisation at
  inference. *Rejected (obs #1953, #1954) — diverges.*
* **Option B:** Keep force-MSE, add soft symmetry penalty
  `λ · ‖K − Kᵀ‖²` to the loss. *Rejected — tested in an earlier
  sprint, gives `‖K−Kᵀ‖/‖K‖ ≈ 0.3` at best; still too asymmetric
  for 5x5 Newton.*
* **Option C:** Predict stress/strain fields and integrate. *Rejected
  — requires per-integration-point supervision we do not have.*
* **Option D:** Predict a scalar potential `Φ(u_bd; coords, h_{t-1})`
  per patch. Derive `F = ∂Φ/∂u_bd` via autograd, `K = ∂²Φ/∂u_bd²`
  via `torch.func.hessian`. Symmetry of `K` is guaranteed by
  Schwarz. *Accepted.*

## Decision Outcome

Chosen option: **D — energy-based (potential) head**.

### Architecture

```
Encoder / Processor / Decoder  →  per-node feature φ_i
                                   │
                                   ▼
                             Φ = sum_i φ_i         (sum pooling, extensive)
                                   │
                                   ▼
                       F = ∂Φ/∂u_bd     (autograd, training + inference)
                       K = ∂²Φ/∂u_bd²   (Hessian, inference only)
```

* Decoder outputs a scalar per node; pooled by `scatter_add` to
  preserve extensivity.
* Training loss: `F_pred = autograd.grad(Φ.sum(), u, create_graph=
  True)[0]`; supervise with MSE against `F_target` in normalised
  coordinates.
* Inference in `surrogate_integrate_material`: route through a new
  `step_functional(u, h_prev)` returning `(Φ, h_new)` that does not
  mutate model state, then `torch.func.hessian` yields `K` directly.
* Hidden state `h_{t-1}` entered as a detached closure constant ⇒
  `K_t` is the correct algorithmic tangent at timestep `t`, matching
  what a classical incremental plasticity solver expects.
* The equilibrium-projection block in
  `graphorge/gnn_base_model/model/gnn_model.py:1057-1069` is gated
  off when `is_energy_head` is active — it would destroy the
  gradient structure.
* Normalisation chain rule: if training uses
  `u_norm = u / L_patch`, the physical tangent is
  `K_phys = (1/L²) · K_norm` (diagonal scaling). `F_phys = (1/L)
  · F_norm`.

### Acceptance criteria (from the plan)

1. **Unit test:** synthetic quadratic `Φ` → `K = K.T` to `1e-10`
   (machine-precision Schwarz check).
2. **Single-sample overfit:** one 5x5 patch trajectory converges
   loss to `< 1e-8` in `< 100` epochs.
3. **Full retrain:** 600 epochs on the existing 5x5 training set
   reaches:
   * `‖K − Kᵀ‖_F / ‖K‖_F < 1e-8` (effectively machine zero).
   * `‖K_gnn − K_analytic‖_F / ‖K_analytic‖_F < 0.5` (vs. current
     `0.96`).
   * Newton-Raphson converges on the reference
     `results/elastoplastic_nlh/2d/quad4/mesh_10x10/patch_5x5/
     n_time_inc_50/` simulation with residual drop ≥ 4 orders per
     increment.

### Positive Consequences

* `K` is structurally symmetric — no post-hoc patches in the FEM
  solver.
* Training is compatible with existing graph dataset; only the
  decoder head and loss change.
* The same head can be used for 1x1, 2x2, 5x5 (sum-pool scales).

### Negative Consequences

* Re-training required for every patch size (`1x1`, `2x2`, `5x5`,
  …) — the legacy force-output checkpoints are not reusable.
* One extra autograd pass per training step to compute `F` from
  `Φ` (≈ 1.5–2× training cost per epoch). Budget allows this.
* Inference requires `torch.func.hessian` which is slower than
  `jacfwd` on the force head. Expected 1.5–3× inference cost per
  patch per iteration — acceptable for correctness.
* `is_energy_head` branches appear across 4 files in 2 repos;
  maintenance burden until the legacy head is removed.
